# ユーザーストーリー

## Epic 1: 初回セットアップ

### US-01: プロジェクトを手動で追加する

**As a** Kiro開発者（タクヤ/ユキ）  
**I want to** フォルダを選択してプロジェクトとして登録する  
**So that** サイドバーから素早くアクセスできるようになる

**Acceptance Criteria**:

```gherkin
Given サイドバーの「Kiro Workspace Hub」ビューが表示されている
When 「プロジェクト追加」アクションをクリックする
Then フォルダ選択ダイアログが表示される

Given フォルダ選択ダイアログが表示されている
When .kiroディレクトリを含むフォルダを選択する
Then プロジェクトがツリービューに追加される
And ~/.kiro-hub/の登録情報が更新される
```

追加チェック:
- [ ] .kiroディレクトリを含まないフォルダを選択した場合、警告を表示して登録可否を確認する
- [ ] 既に登録済みのフォルダを選択した場合、重複メッセージを表示する
- [ ] 存在しないパスは登録できない

**Personas**: タクヤ、ユキ  
**Priority**: MVP

---

### US-02: プロジェクトの自動検出を設定する

**As a** 多くのプロジェクトを持つ開発者（ユキ）  
**I want to** スキャン対象のルートディレクトリを指定して、Kiroプロジェクトを自動検出する  
**So that** 新しいプロジェクトを手動で登録する手間が省ける

**Acceptance Criteria**:

```gherkin
Given 拡張機能の設定画面が開いている
When スキャンパス（ルートディレクトリ）を追加する
Then 指定ディレクトリ配下の.kiroディレクトリを含むフォルダが検出される
And 検出結果がプロジェクト一覧に反映される
```

追加チェック:
- [ ] スキャンはバックグラウンドで非同期実行される
- [ ] スキャン中はプログレス表示がある
- [ ] 検出結果は~/.kiro-hub/にキャッシュされる
- [ ] 手動追加済みのプロジェクトと重複しない

**Personas**: ユキ  
**Priority**: MVP

---

### US-03: 初回起動時のオンボーディング

**As a** 初めて拡張機能を使う開発者（タクヤ/ユキ）  
**I want to** 最初の起動時にプロジェクト追加のガイドを受ける  
**So that** すぐに使い始められる

**Acceptance Criteria**:

```gherkin
Given 拡張機能が初めてアクティベートされた
When サイドバーのツリービューが表示される
Then 「プロジェクトを追加してください」のウェルカムメッセージが表示される
And 手動追加とスキャン設定へのリンクが提示される
```

追加チェック:
- [ ] プロジェクトが1つ以上登録されるとウェルカムメッセージは非表示になる
- [ ] 再度表示する手段は不要（リセットは設定から可能）

**Personas**: タクヤ、ユキ  
**Priority**: MVP

## Epic 2: 日常使用

### US-04: サイドバーからプロジェクト一覧を確認する

**As a** Kiro開発者（タクヤ/ユキ）  
**I want to** サイドバーに登録済みプロジェクトの一覧を見る  
**So that** 利用可能なプロジェクトを把握できる

**Acceptance Criteria**:

```gherkin
Given プロジェクトが1つ以上登録されている
When サイドバーの「Kiro Workspace Hub」ビューを開く
Then 登録済みプロジェクトがツリービュー形式で一覧表示される
And 現在開いているプロジェクトがハイライト表示される
```

追加チェック:
- [ ] 各プロジェクトにプロジェクト名が表示される
- [ ] パスがツールチップまたはサブテキストで確認できる
- [ ] ツリービューの初期ロードは1秒以内

**Personas**: タクヤ、ユキ  
**Priority**: MVP

---

### US-05: プロジェクトを新しいウィンドウで開く

**As a** 複数プロジェクトを並行作業する開発者（タクヤ）  
**I want to** プロジェクトをクリックして新しいKiroウィンドウで開く  
**So that** 現在の作業を中断せずに別プロジェクトにアクセスできる

**Acceptance Criteria**:

```gherkin
Given プロジェクト一覧が表示されている
When プロジェクトをダブルクリック（またはコンテキストメニューから「新しいウィンドウで開く」を選択）する
Then 対象プロジェクトが新しいKiroウィンドウで開かれる
And 現在のウィンドウはそのまま維持される
```

追加チェック:
- [ ] プロジェクトパスが存在しない場合、エラーメッセージを表示する
- [ ] 切り替え操作は2秒以内に完了する

**Personas**: タクヤ  
**Priority**: MVP

---

### US-06: プロジェクトを現在のウィンドウで開く

**As a** ウィンドウを増やしたくない開発者（ユキ）  
**I want to** 現在のウィンドウで別のプロジェクトに切り替える  
**So that** ウィンドウ管理の手間なくプロジェクトを移動できる

**Acceptance Criteria**:

```gherkin
Given プロジェクト一覧が表示されている
When プロジェクトのコンテキストメニューから「現在のウィンドウで開く」を選択する
Then 現在のウィンドウが対象プロジェクトのフォルダに切り替わる
```

追加チェック:
- [ ] 未保存のファイルがある場合、保存確認ダイアログが表示される（VSCode標準動作）
- [ ] 切り替え後、サイドバーのハイライトが更新される

**Personas**: ユキ  
**Priority**: MVP

---

### US-07: プロジェクトを名前で検索する

**As a** 多くのプロジェクトを持つ開発者（ユキ）  
**I want to** プロジェクト名で検索してフィルタリングする  
**So that** 目的のプロジェクトを素早く見つけられる

**Acceptance Criteria**:

```gherkin
Given プロジェクトが複数登録されている
When ツリービューのフィルタ入力欄にテキストを入力する
Then 入力テキストに部分一致するプロジェクトのみ表示される
And マッチしないプロジェクトは非表示になる
```

追加チェック:
- [ ] フィルタはリアルタイム（入力と同時）に反映される
- [ ] フィルタをクリアすると全プロジェクトが再表示される
- [ ] 大文字小文字を区別しない

**Personas**: ユキ  
**Priority**: MVP

---

### US-08: Kiro設定の概要を確認する

**As a** テックリード（ユキ）  
**I want to** 各プロジェクトのKiro設定（hook数、steering数、MCP設定数）を確認する  
**So that** プロジェクトの自動化状況を俯瞰できる

**Acceptance Criteria**:

```gherkin
Given プロジェクトがツリービューに表示されている
When プロジェクトノードを展開する（または詳細ビューを開く）
Then hook数、steering数、MCP設定数が表示される
```

追加チェック:
- [ ] 設定情報は各プロジェクトの.kiro/ディレクトリから読み取る
- [ ] .kiro/ディレクトリにアクセスできない場合は「情報取得不可」と表示
- [ ] 情報は初回表示時にキャッシュし、手動リフレッシュ可能

**Personas**: ユキ  
**Priority**: Post-MVP

## Epic 3: 管理操作

### US-09: プロジェクトにカテゴリ/タグを設定する

**As a** 多くのプロジェクトを管理する開発者（ユキ）  
**I want to** プロジェクトにカテゴリやタグを付ける  
**So that** プロジェクトをグループ化して整理できる

**Acceptance Criteria**:

```gherkin
Given プロジェクトがツリービューに表示されている
When プロジェクトのコンテキストメニューから「タグを編集」を選択する
Then タグ入力のインプットボックスが表示される
And 入力したタグがプロジェクトに保存される
```

追加チェック:
- [ ] 複数タグをカンマ区切りで入力できる
- [ ] 既存タグのサジェスト（オートコンプリート）がある
- [ ] タグでツリービューをグループ化表示できる
- [ ] タグ情報は~/.kiro-hub/に保存される

**Personas**: ユキ  
**Priority**: Post-MVP

---

Then 説明文入力のインプットボックスが表示される

- [ ] 説明文は~/.kiro-hub/に保存される

**Personas**: ユキ  
**Priority**: Post-MVP

### US-11: プロジェクトの登録を解除する

**As a** Kiro開発者（タクヤ/ユキ）  
**I want to** 不要になったプロジェクトを一覧から削除する  
**So that** プロジェクト一覧を整理できる

**Acceptance Criteria**:

```gherkin
Given プロジェクトがツリービューに表示されている
When プロジェクトのコンテキストメニューから「登録解除」を選択する
Then 確認ダイアログが表示される

Given 確認ダイアログが表示されている
When 「はい」を選択する
Then プロジェクトがツリービューから削除される
And ~/.kiro-hub/の登録情報が更新される
And プロジェクトのファイル自体は削除されない
```

追加チェック:
- [ ] 登録解除は~/.kiro-hub/の情報のみ削除（プロジェクトファイルは残る）
- [ ] 解除後もスキャンで再検出される可能性があることを通知に含める

**Personas**: タクヤ、ユキ  
**Priority**: MVP

---

### US-12: プロジェクト名を編集する

**As a** Kiro開発者（タクヤ/ユキ）  
**I want to** プロジェクトの表示名を変更する  
**So that** 分かりやすい名前で管理できる

**Acceptance Criteria**:

```gherkin
Given プロジェクトがツリービューに表示されている
When プロジェクトのコンテキストメニューから「名前を変更」を選択する
Then インラインまたはインプットボックスで名前編集ができる
And 変更した名前がツリービューに即反映される
```

追加チェック:
- [ ] デフォルト名はフォルダ名から自動設定
- [ ] 空の名前は許可しない
- [ ] 名前は~/.kiro-hub/に保存される

**Personas**: タクヤ、ユキ  
**Priority**: Post-MVP

## ストーリー依存関係

```
US-03 (オンボーディング)
  └─→ US-01 (手動追加) ─→ US-04 (一覧確認) ─→ US-05 (新ウィンドウで開く)
  └─→ US-02 (自動検出) ─→ US-04                 └─→ US-06 (現ウィンドウで開く)
                                                   └─→ US-07 (検索)
                             US-04 ─→ US-11 (登録解除)
                             US-04 ─→ US-08 (設定概要) [Post-MVP]
                             US-04 ─→ US-09 (タグ設定) [Post-MVP]
                             US-04 ─→ US-10 (説明設定) [Post-MVP]
                             US-04 ─→ US-12 (名前変更) [Post-MVP]
```
