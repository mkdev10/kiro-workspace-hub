# ユーザーペルソナ

## Persona 1: タクヤ（頻繁切り替え型）

| 項目           | 詳細               |
| -------------- | ------------------ |
| 名前           | タクヤ             |
| 役割           | フルスタック開発者 |
| プロジェクト数 | 3-5個              |
| 切り替え頻度   | 1日に5-10回        |
| 技術レベル     | 中〜上級           |

**特徴**:
- 複数のマイクロサービスプロジェクトを並行して開発
- Kiroの自動化（steering、hook）を各プロジェクトに設定済み
- 切り替えのスピードを最重視
- プロジェクト数が少ないのでタグ/カテゴリは不要と感じるかも

**フラストレーション**:
- 毎回「ファイル > フォルダを開く」でディレクトリを探すのが面倒
- 最近開いたフォルダ一覧から探すのも手間
- ワンクリックで切り替えたい

**ゴール**:
- 最小限の操作でプロジェクト間を行き来する
- よく使うプロジェクトに即座にアクセスする

## Persona 2: ユキ（多プロジェクト管理型）

| 項目           | 詳細                                      |
| -------------- | ----------------------------------------- |
| 名前           | ユキ                                      |
| 役割           | テックリード / プラットフォームエンジニア |
| プロジェクト数 | 15-20個                                   |
| 切り替え頻度   | 週に10-15回                               |
| 技術レベル     | 上級                                      |

**特徴**:
- チームの複数プロジェクトを横断的に見る立場
- プロジェクトをカテゴリ（フロントエンド、バックエンド、インフラ等）で整理したい
- 各プロジェクトのKiro設定状況を俯瞰したい
- 新しいプロジェクトが追加されることが多い

**フラストレーション**:
- プロジェクトが多すぎてどこに何があるか覚えきれない
- プロジェクトの説明やカテゴリがないと探すのに時間がかかる
- 新しく追加されたプロジェクトを見逃すことがある

**ゴール**:
- プロジェクトを体系的に整理・管理する
- 必要なプロジェクトを検索やフィルタですぐに見つける
- 新しいKiroプロジェクトが自動的に検出される
