# 要件定義書: Kiro Workspace Hub

## Intent Analysis

- **ユーザーリクエスト**: Kiroで一般タスクを自動化するプロジェクトを複数扱っているが切り替えが面倒。拡張機能で複数のプロジェクトを横断して切り替えられるようにしたい。
- **リクエストタイプ**: 新規プロジェクト（Kiro専用拡張機能）
- **スコープ**: 複数コンポーネント（サイドバーUI、プロジェクト管理、設定ストレージ、自動検出）
- **複雑度**: 中程度
- **開発言語**: TypeScript

---

## 機能要件（Functional Requirements）

### FR-01: プロジェクト登録

| ID      | 要件                                                                          |
| ------- | ----------------------------------------------------------------------------- |
| FR-01-1 | ユーザーがフォルダパスを手動で追加してプロジェクトを登録できる                |
| FR-01-2 | .kiroディレクトリを含むフォルダを自動検出してプロジェクト候補として提示できる |
| FR-01-3 | 手動追加と自動検出の両方をサポートする                                        |
| FR-01-4 | プロジェクトの削除（登録解除）ができる                                        |

### FR-02: プロジェクト情報管理

| ID      | 要件                                                                      |
| ------- | ------------------------------------------------------------------------- |
| FR-02-1 | 各プロジェクトにプロジェクト名を設定できる                                |
| FR-02-2 | 各プロジェクトにカテゴリ/タグ（ユーザー定義）を設定できる                 |
| FR-02-3 | 各プロジェクトに説明文を設定できる                                        |
| FR-02-4 | 各プロジェクトのKiro設定概要（hook数、steering数、MCP設定数）を表示できる |
| FR-02-5 | プロジェクトのパスを表示する                                              |
| FR-02-6 | プロジェクト情報を編集できる                                              |

### FR-03: プロジェクト切り替え

| ID      | 要件                                                                         |
| ------- | ---------------------------------------------------------------------------- |
| FR-03-1 | サイドバーのツリービューからプロジェクトを選択して切り替えできる             |
| FR-03-2 | 新しいKiroウィンドウで対象プロジェクトを開くオプションを提供する             |
| FR-03-3 | 現在のウィンドウで対象プロジェクトのフォルダに切り替えるオプションを提供する |
| FR-03-4 | ユーザーがウィンドウモード（新規/現在）を選択できる                          |

### FR-04: サイドバーUI（ツリービュー）

| ID      | 要件                                                     |
| ------- | -------------------------------------------------------- |
| FR-04-1 | サイドバーにプロジェクト一覧をツリービュー形式で表示する |
| FR-04-2 | カテゴリ/タグでグループ化して表示できる                  |
| FR-04-3 | プロジェクト名で検索/フィルタリングできる                |
| FR-04-4 | 現在開いているプロジェクトをハイライト表示する           |

### FR-05: 自動検出

| ID      | 要件                                                                        |
| ------- | --------------------------------------------------------------------------- |
| FR-05-1 | 最近開いたKiroプロジェクト（.kiroディレクトリを含むフォルダ）を自動検出する |
| FR-05-2 | 自動検出結果をキャッシュとして~/.kiro-hub/に保存する                        |
| FR-05-3 | 自動検出は設定可能なスキャンパス（ルートディレクトリ）を指定できる          |

### FR-06: データストレージ

| ID      | 要件                                                     |
| ------- | -------------------------------------------------------- |
| FR-06-1 | プロジェクト登録情報を~/.kiro-hub/に一元管理する         |
| FR-06-2 | 各プロジェクトの.kiro/ディレクトリから設定情報を読み取る |
| FR-06-3 | 設定ファイルはJSON形式で保存する                         |

## 非機能要件（Non-Functional Requirements）

### NFR-01: パフォーマンス

| ID       | 要件                                               |
| -------- | -------------------------------------------------- |
| NFR-01-1 | サイドバーの初期ロードは1秒以内に完了する          |
| NFR-01-2 | プロジェクト切り替えは2秒以内に実行される          |
| NFR-01-3 | 自動検出スキャンはバックグラウンドで非同期実行する |

### NFR-02: ユーザビリティ

| ID       | 要件                                                 |
| -------- | ---------------------------------------------------- |
| NFR-02-1 | 直感的なUIで、初回利用でも操作方法が分かる           |
| NFR-02-2 | プロジェクト追加は3クリック以内で完了する            |
| NFR-02-3 | 日本語・英語の両方でUIテキストを提供する（i18n対応） |

### NFR-03: 互換性

| ID       | 要件                                       |
| -------- | ------------------------------------------ |
| NFR-03-1 | Kiro IDE上で動作する拡張機能として開発する |
| NFR-03-2 | VSCode Extension APIを使用する             |
| NFR-03-3 | macOS、Windows、Linuxで動作する            |

### NFR-04: 保守性

| ID       | 要件                                   |
| -------- | -------------------------------------- |
| NFR-04-1 | TypeScriptで開発し、型安全性を確保する |
| NFR-04-2 | モジュール分割された設計とする         |
| NFR-04-3 | ユニットテスト可能な構造とする         |

## 技術的制約

| 項目                 | 値                                                   |
| -------------------- | ---------------------------------------------------- |
| 開発言語             | TypeScript                                           |
| ターゲットIDE        | Kiro（VSCode Extension API互換）                     |
| APIバージョン        | VSCode Extension API (最新安定版)                    |
| データ保存           | ~/.kiro-hub/ (JSON) + 各プロジェクト .kiro/ 読み取り |
| パッケージマネージャ | npm                                                  |
| ビルドツール         | esbuild (VSCode拡張機能標準)                         |
| テストフレームワーク | VSCode Extension Test Framework + Mocha              |

---

## スコープ外

- VSCode Marketplace公開（Kiro専用のため）
- クラウド同期機能
- チーム共有機能
- プロジェクトテンプレート機能
- Git連携（ブランチ情報表示など）
