# ビジネスルール定義

## バリデーションルール

### VR-01: プロジェクトパスのバリデーション
| ルール       | 条件                           | エラーメッセージ                           |
| ------------ | ------------------------------ | ------------------------------------------ |
| 空パス禁止   | path.trim() === ''             | 「パスを入力してください」                 |
| 絶対パス必須 | !path.isAbsolute(path)         | 「絶対パスを指定してください」             |
| パス存在     | !fs.existsSync(normalizedPath) | 「指定されたパスが見つかりません」         |
| 重複禁止     | 正規化パスが既存と一致         | 「このプロジェクトは既に登録されています」 |

### VR-02: プロジェクト名のバリデーション
| ルール     | 条件               | エラーメッセージ                              |
| ---------- | ------------------ | --------------------------------------------- |
| 空名前禁止 | name.trim() === '' | 「プロジェクト名を入力してください」          |
| 最大長     | name.length > 100  | 「プロジェクト名は100文字以内にしてください」 |

### VR-03: タグのバリデーション
| ルール     | 条件              | エラーメッセージ                   |
| ---------- | ----------------- | ---------------------------------- |
| 空タグ禁止 | tag.trim() === '' | （空タグは除外して保存）           |
| 最大タグ長 | tag.length > 50   | 「タグは50文字以内にしてください」 |
| 最大タグ数 | tags.length > 20  | 「タグは20個以内にしてください」   |

### VR-04: 説明文のバリデーション
| ルール | 条件                     | エラーメッセージ                    |
| ------ | ------------------------ | ----------------------------------- |
| 最大長 | description.length > 500 | 「説明は500文字以内にしてください」 |

### VR-05: スキャンパスのバリデーション
| ルール           | 条件                             | エラーメッセージ                           |
| ---------------- | -------------------------------- | ------------------------------------------ |
| 絶対パス必須     | !path.isAbsolute(path)           | 「絶対パスを指定してください」             |
| パス存在         | !fs.existsSync(path)             | 「指定されたスキャンパスが見つかりません」 |
| ディレクトリ必須 | !fs.statSync(path).isDirectory() | 「ディレクトリを指定してください」         |
| 最大パス数       | scanPaths.length > 10            | 「スキャンパスは10個以内にしてください」   |

## ビジネスルール

### BR-01: パス正規化ルール
```
入力パスに対して以下を順次適用:
1. 末尾の '/' または '\' を除去
2. '~' をホームディレクトリに展開
3. fs.realpathSync() でシンボリックリンクを解決
4. パス区切り文字をOS標準に統一
```

### BR-02: プロジェクトIDの生成
```
- UUID v4 形式を使用
- crypto.randomUUID() で生成
- 一度割り当てられたIDは変更不可
```

### BR-03: デフォルト名の決定
```
1. フォルダ名（path.basename(normalizedPath)）をデフォルト名とする
2. ユーザーが明示的に変更しない限り維持
```

### BR-04: スキャン深度制限
```
- 最大深度: 3階層
- 除外ディレクトリ: node_modules, .git, dist, build, out, __pycache__, vendor
- '.' で始まる隠しディレクトリは除外（ただし .kiro の検出自体は行う）
- 最大深度に達したら、そのブランチの探索を打ち切り
```

### BR-05: スキャン結果のマージ
```
- 手動追加済み（source: 'manual'）のプロジェクトと同一パスが検出された場合:
  → スキャン結果を無視（手動登録が優先）
- スキャン検出済み（source: 'scanned'）のプロジェクトと同一パスが再検出された場合:
  → 既存レコードを維持（重複登録しない）
```

### BR-06: 不存在プロジェクトの表示ルール
```
- パスが存在しないプロジェクトもツリービューに表示する
- 警告アイコン（⚠️）を表示
- ツールチップに「パスが見つかりません」と表示
- コンテキストメニューに追加項目:
  - 「パスを更新」→ フォルダ選択ダイアログ
  - 「登録解除」→ 確認後削除
- 切り替え操作は無効化（クリックしてもエラーメッセージのみ）
```

### BR-07: 表示モード切り替え
```
- flat: 全プロジェクトをフラットリストで表示（name昇順ソート）
- grouped: タグ別にグループ化
  - タグなしプロジェクト → 「未分類」グループに配置
  - 複数タグを持つプロジェクト → 各グループに重複表示
  - グループ名はアルファベット/あいうえお順
  - グループ内プロジェクトはname昇順ソート
- 表示モードはsettings.jsonに保存（次回起動時に復元）
```

### BR-08: 現在プロジェクトのハイライト
```
- vscode.workspace.workspaceFolders[0].uri.fsPath を現在プロジェクトパスとする
- 正規化後、登録プロジェクトのパスと比較
- 一致するプロジェクトに contextValue = 'currentProject' を設定
- ツリービューでアクティブアイコンを表示
```

### BR-09: 初回起動判定
```
- StorageService.loadProjects() の結果が空配列 かつ
  settings.json が存在しない（初期化済みフラグなし）
  → オンボーディングを表示
- 一度でもプロジェクト追加されたら、以後オンボーディングは非表示
```

### BR-10: デバウンス書き込みのフラッシュ
```
- 拡張機能deactivate時にflush()を実行
- flush()は保留中の書き込みを即座に実行
- deactivate後の書き込みリクエストは無視
```
